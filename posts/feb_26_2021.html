Recursion in MIPS

MIPS is an assembly language, which is a step down from a high level language like C++. We were asked to implement a recursive algorithm this week for homework using procedures, which are just functions written using universally agreed-upon conventions. This took up most of my week and I must say, I loved every second of it, even the part where I couldn't quite clean my array between inputs. The takeaway of the week is that recursion only works because of the $ra register, or pointer we leave in the '.text' to come back to later. We are effectively copying everything before the recursive call some number of times until we reach the base case. We then go back and finish the function for each copy after the recursive call. Put this way, recursion is simply a way of reorganizing copies of code on the fly. I love this notion, and can't wait to see how it gets implemented at the machine code, then hardware, levels.
