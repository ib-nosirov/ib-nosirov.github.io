Computational thinking

This post is just an introduction to thinking like a computer scientist. It is only meant to familiarize you with the general idea. Computational thinking is a skill and something that needs to be constantly practiced for gradual improvement. I only hope for you to walk away with an appreciation for this way of thinking.

Decomposition: breaking down the problem

In essence, computational thinking is a way of organizing your thoughts so you waste less time staring at the problem. It starts with "what is this question really asking?" For instance, if you were asked to find the roots of the polynomial $3x^2 + 5x + 2$, how would you go about answering that question? If you've seen the problem before, you might factor the polynomial, set it equal to zero, and find the values of $x$ that satisfy the equality. If, on the other hand, you're seeing the problem for the first time, then you need to define some terms. Your first problem is not in performing calculations, but figuring out what all the different parts of a question really mean. In this case, assuming you know what a polynomial is, a quick Google search for 'root of polynomial' turns up the definition: The roots of a polynomial are those values of the variable that cause the polynomial to evaluate to zero. We can then rephrase the original question to ask,

Find the values of $x$ that cause the polynomial $3x^2 + 5x + 2$ to evaluate to zero.

Abstraction: saying it in words first

We can now rewrite the problem as an equality,

$3x^2 + 5x + 2 = 0$ for some number(s) $x$

Finding that number $x$ isn't so easy. There is a $3x^2$ term that will keep increasing, so if we want to get closer to $0$, we need a negative number to equal $x$, that number can't be too negative and it can't be too close to zero... this educated guessing can be very time consuming. We need to come up with some set of steps that will give us the right values every time. Before jumping to writing more equations, computational thinking suggests saying what we want to do in plain words. This is called abstraction and it's the creative bit where details can be ignored and rules can be broken... for now. The goal is to brainstorm some idea. In the example's case, we need to find all the ways that numbers can be added or multiplied to get $0$. We can add negative and positive numbers to get zero and anything times zero is zero. We can work this second angle for a while and come up with factoring, a way of reorganizing a polynomial into a multiplication of two numbers containing $x$, and then finding an $x$ that makes each individual number zero. I'm glossing over this bit because the post is about computational thinking. Check out the video, Factoring algebraic expressions: introduction, for a deeper dive into factoring.

Automation: writing actionable steps

This is where we put our words into sequential steps. Automation itself can be broken down into subproblems and solved using the same computational thinking. In the factoring case, it looks like this $$\begin{align} 3x^2 + 5x + 2 & = 0 \\ (ax + b)(cx + d) & = 0 \\ (3x + b)(1x + d) & = 0 \\ (3x + 2)(x + 1) & = 0 \\ x & = -\frac{3}{2}, -1 \end{align}$$

Analysis: making it better

After coming up with a way of solving the problem, we need to think about finding faults with the method. We need to consider where most of the effort is going, if we can simplify some steps, or if some steps can be shortened. Most importantly, we need to think about when, and how, our solution fails. For instance, what happens to our factoring approach when we can't find numbers that neatly divide or multiply? What if there no real number solutions? Does factoring, as we've defined it, have a way of warning us about these things? Not if, but when, the answer is 'no', it's time to go back to the very first step and resume polishing.

Conclusion: practice makes perfect

So that's it! Different people define computational thinking differently and this is just one such way. Like I said before, I presented a rough outline of what can be going through a computer scientist's head as they are solving a problem. Computational thinking helps us digest new information and think critically about other people's solutions to problems. I doubt many professionals actively think about abstraction-automation-analysis while they are working; over time this just becomes a habitual way of seeing the world.
